Добрый день

Расположение задач в репозитории:

-общая часть: \by\emap\<название уровня из grow или github(например: algorithmization )>\
-далее идет название подуровня в которые группировались задачи на github, если такие были, например: decomposition
-далее, если задача состоит из одного класса, идет файл.java, иначе верхний уровень пакета задачи с последующим разветвлением

-имя каждой задачи выглядит как T(t в случае пакета)<порядковый номер задачи из задания на github>_название задачи: T1_FunctionValue. В случае задачи в пакете(на уровнях кроме последнего(tasks)) такое имя имеет класс, содержащий public static void main(String[] args) (и, иногда, другие методы).

- несколько задач имеют два варианта решения, в таком случае название выглядит как T5A_Name и T5B_Name (так же в курсе была задача с двумя вариантами условия, в этом случае имена после "_" различны);

Пример расположения задачи:
-одним классом:
\by\epam\basicsOfSoftwareCodeDevelopment\linearPrograms\T1_FunctionValue.java

-пакетом:
by\epam\programmingWithClasses\simplestClassesAndObjects\t6_Timer\...


Так же имеются файлы-сохранения, которые имеют разрешения .txt или .ser
